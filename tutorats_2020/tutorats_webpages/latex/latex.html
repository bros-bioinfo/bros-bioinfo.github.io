Tutorats LaTeX







LaTeX, qu'est-ce que c'est ?

C'est un système de composition de haute qualité; il comprend des fonctionnalités conçues pour la production de documentation technique et scientifique. LaTeX est l'un des standards pour la communication et la publication de documents scientifiques. Libre et gratuit, vous pouvez utiliser LaTex facilement. Il est assez aisé de l'installer sur sa machine. Cependant plusieurs éditeurs en ligne existent.

Basé sur un système de balisage, semblable au langage XML (tu verras ça plus tard dans le master ;)), il permet l'écriture de rapport facilement (Bon OK, il faut un peu de bouteille avant que ça soit facile, mais ça vient vite). L'idée est de se soucier que sur le contenu et non sur la forme et le rendu (comme le fait Word).

Le code : partie centrale du LaTeX

Bien, maintenant expliquons un peu le code.

1.  \documentclass{article}
2.  \usepackage[utf8]{inputenc}
3.  
4.  \title{Welcome in Bioinfo}
5.  \author{Bobby le bioinformaticien}
6.  \date{October 2020}
7.  
8.  \begin{document}
9.  
10.  \maketitle
11.  
12.  \section{Introduction}
13.  
14.  \end{document}

Comme dit précédement, LaTeX est un langage de balisage. On retrouve donc la première balise à la première ligne. Cette balise (\documentclass{article}) indique le type de document que l'on souhaite produire en PDF. Ici le type (que l'on peut aussi appeler classe) est article. Or, pour les rapports demandés dans le master, le type de document désiré est report. Donc il faut que tu modifies le type du document ;).

\usepackage permet d'importer un package. C'est un peu comme quand on importe la librairie matplotlib en Python. Entre crochets [ ... ] on met les paramètres et dans les { ... } on nomme le package que l'on veut importer. Ici, on import le package inputenc en donnant comme paramètre utf8. Dit autrement, ce dit que le document utilisera le code utf8.

\title, \author, \date sont des paramètres permettant de donner des informations au document.

\begin{document} et \end{document} sont les balises, qui comme leur nom l'indique, déterminent le début et la fin du document. Tout ce qui est à l'extérieur de ces balises ne sera pas considéré dans le document.

\newpage                     # Débute une nouvelle page
\setcounter{page}{1}              # Réinitialise le compteur de page à 1
\chapter{Les Cours en M1}           # Déclaration du chapitre

\section{IMAJS}
Dans ce cours, l'utilisation du logiciel ImageJ a une grande importance. Il permet de modifer des images en améliorant la netteté ou bien au contraire ajouter du bruit (\textsc{Figure}~\ref{fig:clown}).

\begin{figure}[h]
\centering
\includegraphics[scale=0.4]{clown_Noise-gaussian_Filter_Median-1-3fois.png}

\caption{...} est la légende. \label{...} est le label que l'on donne à la figure. Il sert a faire référence à la figure. En effet, dans le paragraphe au-dessus de la figure, \ref{...} permet de mettre une référence à la figure en qeustion. De plus, quand on clique sur le numéro de la figure, cela nous conduit directement à l'emplacement de la figure. Un lien est présent dans le PDF.

Le [h] permet de fixer la figure à l'endroit où elle est dans le code. H pour "Here". Le "~" entre "\textsc{Figure}" et "\ref{fig:clown}" représente un espace insécable.

Ce paragraphe était présent :

To estimate the true substitutions number, we calculate the Hamming distance between the initial sequence and the mutate sequence created. For that, we have used the definition of the distance between two string given by R. Hamming [1]. While comparing two strings of equal length, Hamming distance is the number of positions in which the two nucleotides are different.

Pour citer des références on peut utiliser un bibliographie contenue dans un fichier externe. Heureusement Bobby avait sauvegardé celui-ci, ICI. Il faut ensuite insérer la bibliographie au document avant de pouvoir citer des références. Pour cela, le package biblatex peut être utilisé :

\usepackage[ backend=biber, style=ieee, citestyle=ieee ]{biblatex} \addbibresource{biblio.bib}

Tu sais sûrement que plusieurs formats de citation existent. Ici, on utilise le format "IEEE". À la fin du document, il faut insérer la ligne :

\newpage
\addcontentsline{toc}{chapter}{Bibliographie} # Ajoute la biblio au sommaire
\printbibliography

Pour pouvoir citer la référence dans le texte, il suffit de faire \cite{nom_reference} où l'on veut avoir la citation. A note que le nom de la référence est sur la première ligne de la référence. Ici, on a @article{hamming_error_1950,, donc le nom de la référence est "hamming_error_1950". Tu remarqueras dans le PDF que tu peux cliquer sur le numéro de la citation. Cela t'ammènera à la référence dans la bibliographie. A noter que d'autres package existe pour gérer la bibliographie.

Programmation

Voici la fonction que Bobby voulais insérer dans son rapport pour cette partie :

def hamming(seq1, seq2):
  """
  Calculate the hamming distance between two sequences passed in arguments
  """
  hamming_distance = 0
  # If sequences have not the same size, the calcul of hamming distance is impossible
  if len(seq1) != len(seq2):
    print("Error : Length of the two sequences are not the same.\n")
  # Convertion of the two sequence string to lists
  seq1_list = list(seq1)
  seq2_list = list(seq2)
  # For each nucleotides, verification of the equality between the two sequences
  for i in range(len(seq1)):
    # If no equality, put +1 to the hamming distance
    if seq1_list[i] != seq2_list[i]:
      hamming_distance += 1
  return hamming_distance

Le but ici est d'insérer cette fonction au document LaTeX. Pour cela des packages existent tels que listing ou minted. Nous allons utiliser le dernier. Deux possibilités existent : soit mettre le code en "dur" dans le document LaTeX, soit utiliser un fichier que le document LaTeX va lire. Pour la dernière méthode, tu peux faire comme ça :

\subsection{Programmation}
  Python le fameux serpent !
  \inputminted{python}{hamming_function.py}

Ici, on importe via la commande \inputminted, le contenu du fichier "hamming_distance.py". En indiquant que c'est du Python, la mise en forme sera réalisée sur le PDF.
